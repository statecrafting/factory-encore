# {{APP_NAME}}

Dual-app monorepo: two independent Encore.ts applications sharing a repository.

| App | Audience | Auth Driver | Port (dev) |
|-----|----------|-------------|-----------|
| `public/` | External users | rauthy OIDC | 4000 |
| `internal/` | Staff | rauthy OIDC | 4001 |

Each app is a **standalone Encore.ts application** with its own Gateway, authHandler, stateless RS256 JWT, `SQLDatabase("app")`, secrets, and deploy pipeline. The two apps share no backend code at runtime; their trust zones (external user vs staff, each with its own rauthy registration) are kept hard.

> **Backend = Encore.ts.** This app was generated from the Vue + Encore Template.
> The Express `api-public`/`api-internal` server model and Redis/PostgreSQL session stores are retired.

## Prerequisites

- Node.js 24.x or higher
- npm 10.x or higher
- [Encore CLI](https://encore.dev/docs/install)
- Docker (Encore provisions a local Postgres per app during `encore run`)

## Quick Start

```bash
# Install SPA packages in each app root
cd public && npm install && cd ..
cd internal && npm install && cd ..

# Install each Encore backend and generate JWT keys
cd public/apps/api && npm install && npm run generate-keys && cd ../../..
cd internal/apps/api && npm install && npm run generate-keys && cd ../../..

# Run the public app (api :4000, web :5173)
cd public && npm run dev

# In a separate terminal: run the internal app (api :4001, web :5174)
cd internal && npm run dev:api -- --port=4001   # or configure PORT in .env
```

- **Public web**: http://localhost:5173  (external user-facing)
- **Internal web**: http://localhost:5174  (staff-facing)
- **Public API**: http://localhost:4000
- **Internal API**: http://localhost:4001

The Vite dev servers proxy `/api/*` to their respective Encore API port. Docker must be running so Encore can start each app's local Postgres database.

### Development Auth

Both apps default to `AUTH_DRIVER=mock` in dev: no external IdP required. Use `?user=0|1|2` on the login URL for each app.

## Key Environment Variables

**Public app (`public/.env`):**

```bash
AUTH_DRIVER=rauthy

# rauthy OIDC (self-hosted provider; public client registration)
RAUTHY_ISSUER=<rauthy issuer URL>
RAUTHY_CLIENT_ID=<public client ID>
RAUTHY_CLIENT_SECRET=<public client secret>
RAUTHY_REDIRECT_URI=https://<your-public-domain>/api/v1/auth/rauthy/callback
RAUTHY_DEFAULT_ROLE=user
FRONTEND_URL=https://<your-public-domain>

# API Gateway (BFF proxy; optional)
PRIVATE_API_BASE_URL=<private backend URL>
GATEWAY_OAUTH_TENANT_ID=<OAuth tenant/realm>
GATEWAY_OAUTH_CLIENT_ID=<client ID>
GATEWAY_OAUTH_CLIENT_SECRET=<client secret>
GATEWAY_OAUTH_SCOPE=<scope>
```

**Internal app (`internal/.env`):**

```bash
AUTH_DRIVER=rauthy

# rauthy OIDC (self-hosted provider; internal client registration)
RAUTHY_ISSUER=<rauthy issuer URL>
RAUTHY_CLIENT_ID=<internal client ID>
RAUTHY_CLIENT_SECRET=<internal client secret>
RAUTHY_REDIRECT_URI=https://<your-internal-domain>/api/v1/auth/rauthy/callback
RAUTHY_DEFAULT_ROLE=user
FRONTEND_URL=https://<your-internal-domain>

# API Gateway (BFF proxy; optional)
PRIVATE_API_BASE_URL=<private backend URL>
GATEWAY_OAUTH_TENANT_ID=<OAuth tenant/realm>
GATEWAY_OAUTH_CLIENT_ID=<client ID>
GATEWAY_OAUTH_CLIENT_SECRET=<client secret>
GATEWAY_OAUTH_SCOPE=<scope>
```

JWT keys for each app are declared as Encore secrets in `apps/api/infra.config.json`. In dev they are generated by `npm run generate-keys` (inside `apps/api`) and live in `apps/api/keys/*.pem`.

See `docs/AUTH-SETUP.md` for full rauthy configuration details.

## Commands

Run these from inside the relevant app root (`public/` or `internal/`):

```bash
# Development
npm run dev              # api (encore run) + web/web-internal, concurrently
npm run dev:api          # Encore API only
npm run dev:web          # SPA only

# Build
npm run build            # shared packages + SPA (emits into apps/api/web/build)
npm run build:api        # backend image: encore build docker --base (run from apps/api)

# Testing
npm test                 # vitest across workspaces
npm run test:e2e         # Playwright E2E tests
npm run typecheck        # type-check SPA + packages
npm run typecheck:api    # encore check (backend graph + topology + types)

# Quality
npm run lint
npm run lint:fix
npm run format

# Backend helpers (run inside apps/api)
npm run generate-keys    # RSA-2048 JWT signing keys → apps/api/keys/*.pem
npm run gen:client       # regenerate typed client
```
