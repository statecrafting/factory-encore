# {{APP_NAME}}

Public-facing application: Vue 3 SPA + Encore.ts BFF API with rauthy OIDC auth, stateless RS256 JWT, Postgres, and OAuth S2S API gateway.

> **Backend = Encore.ts.** This app was generated from the Vue + Encore Template.
> `apps/api` is a standalone Encore app; it is excluded from npm workspaces and has its own lockfile.

## Prerequisites

- Node.js 24.x or higher
- npm 10.x or higher
- [Encore CLI](https://encore.dev/docs/install)
- Docker (Encore provisions a local Postgres for `SQLDatabase("app")` during `encore run`)

## Quick Start

```bash
# 1. Install the SPA + shared packages (npm workspaces)
npm install

# 2. Install the standalone Encore app and generate dev JWT keys
cd apps/api
npm install
npm run generate-keys       # writes apps/api/keys/*.pem (gitignored)
cp .env.example .env        # optional; dev fallbacks work without it
cd ../..

# 3. Run everything (api on :4000, web on :5173)
npm run dev
```

- **Web (public)**: http://localhost:5173
- **API**: http://localhost:4000
- **Health**: http://localhost:4000/health

`npm run dev` builds the shared packages, then starts the Encore API (`encore run --port=4000`) and the Vue dev server concurrently. The Vite dev server proxies `/api/*` to port 4000. Docker must be running so Encore can start the local Postgres database.

### Development Auth

The default dev config uses `AUTH_DRIVER=mock` with no external identity provider required. Use `?user=0|1|2` on the login URL to switch mock users.

To test rauthy locally, set `AUTH_DRIVER=rauthy` and configure the rauthy env vars (see `docs/AUTH-SETUP.md`).

### Key Environment Variables (Production)

```bash
# Auth driver
AUTH_DRIVER=rauthy

# rauthy OIDC (self-hosted provider)
RAUTHY_ISSUER=<rauthy issuer URL>
RAUTHY_CLIENT_ID=<your client ID>
RAUTHY_CLIENT_SECRET=<your client secret>
RAUTHY_REDIRECT_URI=https://<your-domain>/api/v1/auth/rauthy/callback
RAUTHY_DEFAULT_ROLE=user     # role assigned when the provider sends no role claims

# JWT signing keys (declare as Encore secrets in infra.config.json for production)
# In dev these are generated by npm run generate-keys and live in apps/api/keys/*.pem
# JWT_PRIVATE_KEY, JWT_PUBLIC_KEY, JWT_REFRESH_PRIVATE_KEY, JWT_REFRESH_PUBLIC_KEY

# Frontend URL (redirect target after login)
FRONTEND_URL=https://<your-domain>

# API Gateway (BFF proxy; gateway returns 503 if unset)
PRIVATE_API_BASE_URL=<private backend URL>
GATEWAY_OAUTH_TENANT_ID=<OAuth tenant/realm>
GATEWAY_OAUTH_CLIENT_ID=<service principal client ID>
GATEWAY_OAUTH_CLIENT_SECRET=<service principal secret>
GATEWAY_OAUTH_SCOPE=<private API scope>
```

See `docs/AUTH-SETUP.md` for full rauthy configuration and `infra.config.json` for the Encore secret binding model.

## Commands

```bash
# Development
npm run dev              # api (Encore :4000) + web, concurrently
npm run dev:api          # Encore API only (encore run --port=4000)
npm run dev:web          # public SPA only

# Build
npm run build            # shared packages + web SPA (emits into apps/api/web/build)
npm run build:api        # backend image: encore build docker --base (run from apps/api)

# Testing
npm test                 # vitest across workspaces
npm run test:e2e         # Playwright E2E tests
npm run typecheck        # type-check the SPA + packages
npm run typecheck:api    # encore check (backend graph + topology + types)

# Quality
npm run lint
npm run lint:fix
npm run format

# Backend helpers (run inside apps/api)
npm run generate-keys    # RSA-2048 JWT signing keys → apps/api/keys/*.pem
npm run gen:client       # regenerate typed client → apps/web/src/lib/encore-client.ts
```
